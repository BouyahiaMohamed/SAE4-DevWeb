KDOU nous a fourni une base de données avec plusieurs tables :

  • Fournisseur

Notre équipe s'est donc interresé à cette table. C'était une phase de découverte. Nous avons donc fait quelques requêtess pour nous familiariser avec cette base de données.

Qu'est-ce que le QBE ?

Le QBE, ou Query By Example est un type d'interface utilisateur permetant de faire des requêtes dans une base de données sans écrire de code (le code en langage SQL est généré). Cette outil offre une simplicité d'utilisation qui nous a permi de nous concentrer sur la base de données et non sur la syntaxe des requêtes SQL.

Plusieurs applications nous permettent de faire du QBE, nous avons choisi l'application Access, développée par Microsoft.


L'interface QBE de Kexi.

Un probleme se pose

Mais un probleme va vite s'imposer. Nous nous sommes demandé s'il y a, dans la clientèle, des clients qui n'ont pas passé de commande.

On a malgré tout réussi à répondre à cette question,
voici la liste des clients qui n'ont pas passé de commande : ATAUG, AUBBA, CONTIG, GOURL, GOUTA, HOTLA.

Mais il était compliqué de répondre à cette question car il fallait comparer la liste des clients et la liste des commandes. Les erreurs sont simples à faire avec cette méthode.


C'est à ce moment qu'on a compris que la base de données n'est pas parfaite.
On peut voir que dans la table Fournisseur des pays ont des noms différents d'un enregistrement à un autre, par exemple les États-Unis ont été écrits de trois manières différentes : “Etats Unis”, “Etats-Unis” et “USA”, ce qui peut rendre compliqué la recherche. Il y a des cases vides dans plusieurs colonnes, ce qui n'est pas possible normalement dans une base de données. Les formats des numéros de téléphone sont tous différents, certains ont le code des pays et d'autres non. Dans la table clientèle, les noms des villes sont écrits à chaque enregistrement et donc des fois ils sont mal écrits, certaines villes ont des majuscules et des fois non, d'autres des tirets ou pas. La fonction des Fournisseurs peut aussi être mal orthographiée sur certains enregistrements. Dans quantité par unité, il y a plein d'unités différentes, ce qui peut porter à confusion


Nous pouvons donc améliorer l'organisation des données en instaurant une cohérence entre les tableaux. Pour ce faire, nous pouvons utiliser l'application Access. Ainsi, les données seront liées entre elles,et il n'y aura aucun problème si on efface, supprime ou ajoute des clients ou produits. Il faut aussi créer de nouvelles tables pour les pays ou pour les méthodes de conditionnement. Il faut aussi mettre en place des conventions de nommages pour les addresses et les numéros de téléphone.


Conclusion


Pour KDou, nous recommandons d'utiliser l'application Access au lieu de Excel pour une meilleure organisation des données et pour éviter les erreurs. De plus, en optimisant l'accès aux données, le temps de travail s'en verra réduit et par conséquent, la consommation électrique sera réduite, ce qui limitera la pollution et le coût. Ainsi, l'entreprise fera des économies budgétaires tout en étant plus “éco-friendly”.

En revanche, l'utilisation d'Access, ou plus généralement d'une base de données, nécessite une certaine rigeur. Ainsi, notre équipe peut aider KDOU à mieux organiser leurs informations pour que la base de données reste cohérente dans tous les cas possibles.
